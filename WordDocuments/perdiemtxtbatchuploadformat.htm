When importing this data, the system breaks down local into breakfast, lunch, dinner, and incidentals according to the percentages specified in the KFS CONUS M&IE Breakdown record for the applicable total. If no CONUS M&IE Breakdown record contains the total specified in the CONUS record, the system uses the value specified in your institution’s DEFAULT_CONUS_MIE_BREAKDOWN parameter to obtain the percentages applied to breakfast, lunch and dinner.

The fields of each OCONUS and foreign record are (in order):

•      country_code

•      country_name

•      conus_ind

•      location_name

•      time_zone

•      linked _country_code

•      linked_location_name

•      dod_ind

•      eff_date

•      exp_date

•      start_date

•      end_date

•      lodging_rate

•      local_meals

•      meals_rate_only

•      proportional_meals_rate

•      incidentals

•      footnote_ref

•      footnote_rate

•      max_per_diem

When importing OCONUS and foreign data:

•      The system uses the Meals Rate Only field and the Incidentals fields.

•      The system breaks down the Meals Rate Only value into breakfast, lunch and dinner based on the values stored in your institution’s OCONUS_MEALS_BREAKDOWN parameter (for example, 15;25;40, which represent the percentages for breakfast, lunch and dinner, respectively). 

•      If the country code does not identify the United States, the system loads the OCONUS rate as International trip type.

•      If the country code identifies the country as the United States, the system loads it as either in-state or out-of-state depending on the setting of your institution’s INSTITUTION_STATE parameter.

After successfully uploading the CSV file, a barcode inventory error document (BCIE) will be created when scanned records do not pass validity checks against the asset database. The BCIE document is dropped into the initiator’s action list.

 

Adding a barcode inventory file causes the CAM module to display the number of records uploaded and the number of records in error. CAM also identifies by document number the error report created for this file upload.

Chart of Accounts

 >  >

The KFS relies on a variety of tables to function. These tables define the attributes and values that the system uses to validate transactions. They also control the often complex relationship between elements in the Chart of Accounts (COA) that need to be leveraged for internal and external reporting. For example, the KFS needs to store basic information such as the sub-fund groups to which new accounts may be assigned or the calendar month that correlates to the first month of the institution's fiscal year. This type of flexibility is one of the features that make the KFS such a configurable application. This section introduces key concepts relating to maintaining the reference tables using the COA maintenance documents accessed via the Main Menu tab.

The Chart of Accounts submenu group provides access to several different types of COA documents.

   For more information on global documents mentioned below, see Global Documents.

Chart of Accounts submenu group

Document Type

 

Description

Account (ACCT)

Identifies a pool of funds assigned to a specific university organizational entity for a specific purpose. Global document functionality is available for this document.

Account Delegate / Model (ADEL/GDLM)

A KFS user that is authorized under certain circumstances to take workflow actions on a document in the place of an account's Fiscal Officer. Global and Global Model document functionality are available for this document.

Object Code  (OBJT)

Represents classifications that are assigned to transactions and helps identify the types of transaction, including income, expense, asset, liability, and fund balance. Global document functionality is available for this document.

Organization Code  (IRGB)

Represents units at many different levels within the institution such as department, responsibility center, and campus

Project Code  (PROJ)

Identifies particular transactions that might span multiple accounts

Sub-Account  (SACC)

An optional element of the accounting string that allows you to track financial activity within a particular account at a finer level of detail

Sub-Object Code  (SOBJ)

An optional element of the accounting string that allows you to create finer distinctions within a particular object code for a specific account. Global document functionality is available for this document.

   Additional COA documents maintained by the central administration office are covered in the Guide to the KFS Maintenance Menu and Maintenance Menu help.
